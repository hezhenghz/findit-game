# PRD: FindIt — 儿童"找一找"游戏

## Problem Statement

一个爸爸想给自己的孩子做一个有趣的"找一找"游戏——在一个丰富的场景画面中找到指定物品。但他没有美术资源，需要靠 AI 自动生成关卡；同时他希望把关卡质量控制在自己手里，只让孩子玩经过筛选的内容。长期来看，他希望把网站开放给更多人，形成用户贡献 AI 资源、社区审核关卡的互助模式。

## Solution

FindIt 是一个 Web 端的儿童寻物游戏。爸爸在管理后台用自然语言描述场景，AI（火山引擎 Seedream + Doubao Vision + DeepSeek）自动生成关卡底图、识别目标物品坐标、裁切小图。爸爸审核挑选后，关卡入库。孩子打开游戏页面，看到底部任务栏里 5 个目标物品的小图，在场景大图中逐一找到并点击，找齐即通关。

## User Stories

### 管理后台（爸爸）

1. 作为管理员，我想要点击"增加关卡"按钮进入创建流程，以便开始制作新关卡
2. 作为管理员，我想要输入几个关键词（如"女孩、甜品"或"汪汪队"），点击"AI 补充"按钮，让 AI 自动生成详细的场景描述和目标物品清单，以便节省手动编写的时间
3. 作为管理员，我想要在 AI 生成的描述和物品清单上手动修改，以便调整到我满意的效果
4. 作为管理员，我也可以完全跳过 AI 补充，手动填写场景描述和物品类型，以便在已有想法时直接创作
5. 作为管理员，我想要通过 1-5 星滑块设置关卡难度，必要时展开高级设置面板调整每个量化的难度参数（目标物品大小占比、干扰物数量、画面总物品数），以便精确控制难度
6. 作为管理员，我想要点击"生成"按钮后，看到异步进度提示，系统在后台同时生成 4 张候选场景底图，以便从中挑选最好的一张
7. 作为管理员，我想要在生成完成后看到 4 张候选图，每张图上标注出 AI 识别到的物品位置（bounding box），以便判断标注准确性
8. 作为管理员，我想要手动调整某个物品的标注框位置，以便纠正 AI 标注不准的地方
9. 作为管理员，我想要选择接受一张或多张候选图，被接受的自动生成裁切小图并入库，以便关卡进入可玩状态
10. 作为管理员，我想要拒绝不满意的候选图，以便重新生成
11. 作为管理员，我想要进入"关卡库"页面浏览所有已入库的关卡（按缩略图展示），以便回顾已有内容
12. 作为管理员，我想要在关卡库中按关键词或标签搜索关卡，以便快速找到特定关卡
13. 作为管理员，我想要在关卡库中预览关卡的完整信息（大图、物品名称和位置），以便确认内容正确
14. 作为管理员，我想要在关卡库中删除不再需要的关卡，以便保持内容整洁

### 游戏端（孩子）

15. 作为小玩家，我想要进入游戏后看到一个关卡选择页面，关卡以缩略图网格展示，以便看图选择想玩的关卡
16. 作为小玩家，我想要关卡按顺序解锁（通过前一关才能玩下一关），未解锁的关卡显示为锁定状态，以便有成就感和动力
17. 作为小玩家，我想要点击选择一关进入游戏，看到上方大图中下方任务栏的布局，任务栏显示 5 个要找的物品小图和名称，以便对照寻找
18. 作为小玩家，我想要在上方大图中点击我认为是目标物品的位置，系统判断是否命中，以便体验寻找的乐趣
19. 作为小玩家，当我点中了目标物品时，我想要物品对应的任务栏图标被划掉并播放"找到"动画，以便清楚知道还有几个要找
20. 作为小玩家，当我点错（没有点到任何目标物品的判定区域）时，我希望没有任何惩罚，只是简单地没有反应，以便能放心探索
21. 作为小玩家，当 5 个物品全部找到后，我想要看到一个通关庆祝动画，以便获得成就感
22. 作为小玩家，通关后我想要自动解锁下一关，以便继续挑战

### 系统

23. 作为系统，我需要接受管理员的关卡生成请求后异步执行 AI 流水线（prompt 拼接→Seedream 生成底图→Doubao Vision 标注坐标→裁切小图→标签提取），并提供进度状态供前端轮询

## Implementation Decisions

### 平台与框架
- **平台**：Web 浏览器
- **前端框架**：Next.js（React + API Routes）
- **游戏渲染**：PixiJS 负责游戏画布，React 负责 UI 外框（任务栏、工具栏、弹窗）
- **数据库**：SQLite + Drizzle ORM（MVP），后续迁移到 Postgres
- **AI 服务**：火山引擎 Seedream（文生图）、Doubao Seed 2.0 Lite（视觉定位 Grounding）、DeepSeek（文本扩充 + 标签提取）
- **图片存储**：MVP 阶段本地 `public/` 目录，后续迁移到火山引擎 TOS

### 架构分工
- PixiJS 负责：场景大图渲染、点击坐标接收、命中判定触发、找到物品的动画效果
- React 负责：底部任务栏（物品小图 + 名称 + 已找到状态）、顶部信息栏、通关弹窗、关卡选择页
- React 持有全部游戏状态（找到多少个、还剩几个），PixiJS 只渲染和发送点击事件
- 两层的桥接通过自定义事件/回调：PixiJS 检测命中 → emit 事件 → React 更新状态

### 图片模式
- 物品直接画在场景底图中（嵌入模式），非独立叠加
- 目标物品的小图从底图按 bounding box 裁切，保证小图和底图中物品完全一致

### 图片规格
- 游戏内所有场景图统一固定尺寸（2048x2048），bounding box 坐标直接基于该标准画布

### 关卡数据结构
- 两张表：`difficulty_presets`（难度预设）和 `levels`（关卡）
- `targetItems` 作为 JSON 数组内嵌在 `levels` 行中，不拆表（物品和关卡严格绑定，不存在复用关系）
- AI 生成时自动提取 `tags`（结构化分类标签），用于关卡库搜索

```typescript
// difficulty_presets 表
{
  id: number;
  level: number;           // 1-5 星
  targetSizePercentMin: number;
  targetSizePercentMax: number;
  interferenceCountMin: number;
  interferenceCountMax: number;
  totalItemCountMin: number;
  totalItemCountMax: number;
  updatedAt: Date;
}

// levels 表
{
  id: number;
  title: string;
  keywords: string;
  tags: string;
  sceneDescription: string;
  difficulty: number;      // 1-5
  sceneImage: string;      // 底图路径
  status: 'draft' | 'approved';
  targetItems: [
    {
      name: string;
      cropImage: string;
      bboxXMin: number;
      bboxYMin: number;
      bboxXMax: number;
      bboxYMax: number;
      sortOrder: number;   // 1-5
    }
  ];
  createdAt: Date;
}
```

### AI 管线
- 异步执行 + 前端轮询状态（非 SSE/WebSocket）
- 一次生成同时产出 4 张候选底图
- 流程：用户输入关键词 → DeepSeek 补充场景描述 + 物品清单 → 用户确认 → 拼接含难度参数的 prompt → Seedream 生成 4 张底图 → Doubao Vision 标注每张图的物品坐标 → 裁切小图 → DeepSeek 提取 tags → 展示给用户审核

### 难度系统
- 难度参数全局预设（`difficulty_presets` 表），修改后新生成的关卡自动使用
- 星级滑块 + 可展开的高级参数面板（C 方案）
- 量化的难度参数写入 Seedream prompt，由 AI 尽力遵循，最终难度由人工审核把关

### 游戏交互
- 点击模式：孩子点击大图中目标位置，系统判定坐标是否命中 bounding box
- 无惩罚机制：点错无反应，不限次数，无计时
- 通关条件：5 个物品全部找到 → 庆祝动画 → 自动解锁下一关

### 关卡解锁
- 玩家端：关卡选择页 + 线性解锁（通关解锁下一关）

## Testing Decisions

### 测试原则
- 只测试外部行为，不测试实现细节
- AI 管线调用（第三方 API）使用 mock，测试编排逻辑
- 游戏引擎的点击判定用纯逻辑测试，不依赖 Canvas 渲染

### 各模块测试内容

| 模块 | 测试内容 |
|------|----------|
| 数据库层 | Schema 正确性、CRUD 操作、种子数据完整性 |
| AI 管线 | Prompt 拼接正确性（含难度参数映射）、异步流程编排逻辑、API 响应解析、错误处理 |
| 管理后台 API | 各端点请求/响应、异步任务状态流转、参数校验 |
| 管理后台前端 | 表单交互流程、AI 扩充/生成按钮行为、预览图选择、关卡库搜索/删除 |
| 游戏引擎 | 点击坐标命中判定（命中/未命中/边界）、游戏状态机（未开始→进行中→完成）、找到物品后的状态变化 |
| 游戏前端 | 关卡选择页锁定/解锁逻辑、游戏页 PixiJS↔React 事件桥接、任务栏状态变化、通关弹窗触发 |

## Out of Scope

- 用户注册/登录系统
- 金币系统（获取、消耗、交易）
- 用户贡献 API Key 功能
- 社区众包审核
- 云存储（TOS/OSS）
- 移动端 App / 小程序
- Postgres 数据库
- 联网对战或排行榜
